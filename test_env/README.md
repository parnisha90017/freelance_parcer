# Test Environment

## Запуск

```bash
python test_env/test_bot.py
```

## Зачем это нужно
- для тестирования новых фич без остановки основного бота
- чтобы безопасно проверять изменения на отдельной базе и в отдельных логах
- чтобы отлаживать Groq AI-оценку и генерацию откликов
- чтобы проверять новые парсеры перед подключением к основному боту

## Что сейчас тестируется
- тестовый бот
- Telegram channels parser
- standalone парсеры источников и API-контуров

## Боты
- Основной бот: @dizai_kwork_parser_bot
- Тестовый бот: @dizai_parser_test_bot

## Для себя
- Сначала проверяй новые изменения здесь, потом в основном боте.
- Если ломается логика уведомлений или callback'ов, сначала смотри `logs/test_parser.log`.
- Не путай `data/test_projects.db` с основной базой.
